
# Phishing Awareness Project using GoPhish

## Project Overview

As part of my role, I contributed to a phishing awareness initiative designed to simulate real-world phishing scenarios in a controlled environment.
The goal was to understand how employees respond to phishing emails and improve overall cybersecurity awareness.

Key behaviors observed:
- Clicking suspicious links
- Reporting phishing attempts

## Introduction

Cyber threats continue to evolve, but many still rely on human interaction. Common attack vectors include:
- Malware – damages systems or steals data
- Ransomware – locks files until a ransom is paid
- Social Engineering – manipulates users to gain access

Phishing, a common form of social engineering, tricks users into revealing sensitive information by impersonating trusted sources.

This project uses GoPhish to simulate phishing scenarios in a controlled and authorized environment. The objective is to measure awareness, educate users, and strengthen security posture.

## Project Workflow

The project emphasizes process and analysis rather than tool installation:

- Created safe and realistic phishing scenarios
- Designed non-malicious email templates
- Configured sending profiles in a lab environment
- Launched authorized simulations
- Monitored activity via dashboard
- Analyzed user behavior

## Privacy-First Design

This project prioritizes user privacy and ethical practices:
- No collection of sensitive data (e.g., passwords, personal info)
- Only essential metrics tracked (click, report & open rate)
- Focus on awareness, not punishment
- Conducted with proper authorization
