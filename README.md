## Project Overview
As part of my role, I contributed to a phishing awareness initiative designed to simulate real-world phishing scenarios in a controlled environment.
The main goal was to understand how employees respond to phishing emails and to improve overall cybersecurity awareness. 

Key behaviors observed included:
- Clicking suspicious links
- Reporting phishing attempts

## Introduction

Cyber threats continue to evolve, but many attacks still rely heavily on human interaction. Common attack vectors include:
- Malware – software that can damage systems or steal data
- Ransomware – software that locks systems or files until a ransom is paid
- Social Engineering – manipulating human behavior to gain access or information

Phishing, a highly effective type of social engineering, tricks users into revealing sensitive information like login credentials or financial details depending on how the threat actor delivers the attacks.
In this project, I used GoPhish to simulate phishing scenarios in a contrlled lab and has been approved by management. The purpose is to measure user awareness, educate employees abd strengthen organizational cybersecurity posture.

🛠️ Tool Used

**GoPhish** – is a powerful, easy-to-use, open-source phishing toolkit meant to help pentesters and businesses conduct real-world phishing simulations.

## Project Workflow

The project emphasizes process and analysis rather than tool installation:
- Created safe, realistic phishing scenarios
- Designed generic, non-malicious email templates
- Configured sending profiles in a controlled lab
- Organized users into test groups
- Launched authorized phishing campaigns
- Monitored user activity via the GoPhish dashboard
- Analyzed behavioral patterns and outcomes

## Privacy-First Design

This project follows a privacy-first approach, ensuring user trust and data protection were prioritized:
- No Sensitive Data Collection – No credentials, personal data, or form submissions are captured
- Minimal Data Tracking – Only essential metrics (click rate and report rate) were used
- User Protection First – Focused on education, not penalization
- Authorized & Controlled Execution – All activities approved by management
- Ethical Security Practices – Fully aligned with responsible cybersecurity standards

Emphasizes privacy-by-design, ethical security testing, and responsible data handling.

## Metrics & Analysis

The following metrics were tracked to evaluate effectiveness:
- Click Rate – Users who clicked phishing links
- Report Rate – Users who reported suspicious emails

⚠️ Note: No sensitive user data, credentials, or submissions were collected.

Key Insights:
- Some users interacted with links, highlighting awareness gaps
- Reporting behavior varied, emphasizing the need for training
- Behavioral trends helped guide targeted educational initiatives
- Outcome:
- Provided actionable insights for improving security awareness
- Reinforced the importance of user education
- Maintained user privacy while still evaluating risk 

## Purpose
The project aims to strengthen the human layer of cybersecurity by:
- Raising awareness of phishing tactics
- Encouraging critical thinking before interacting with emails
- Promoting a culture of reporting suspicious activity
- Reducing the likelihood of successful cyberattacks

## Security & Ethical Considerations

- Awareness training significantly improves user response
- Reporting mechanisms are critical for early threat detection
